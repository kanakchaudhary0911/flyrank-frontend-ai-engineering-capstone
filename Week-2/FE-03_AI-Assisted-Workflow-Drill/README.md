# FE-03 – AI-Assisted Workflow Drill

## Overview

This assignment explores the impact of prompt quality on AI-assisted software development by implementing the same feature using two different prompting approaches.

The objective is to compare a minimal prompt with a structured AI workflow and evaluate their differences in code quality, accessibility, testing, edge-case handling, and review effort.

---

## Objectives

- Understand the importance of effective prompt engineering.
- Compare vague and structured AI prompts.
- Practice an AI-assisted development workflow.
- Verify AI-generated code through testing and review.
- Document the complete development process.

---

## Assignment Workflow

### Round 1 – Vague Prompt

- Used a single, minimal prompt.
- Accepted the generated output with minimal guidance.
- Saved the implementation in a dedicated Git branch.

### Round 2 – Structured Prompt

- Started with a fresh AI session.
- Used a detailed prompt including:
  - Project context
  - Technical constraints
  - Accessibility requirements
  - Expected behavior
  - Verification instructions
- Followed an Explore → Plan → Implement → Test → Review workflow.
- Saved the improved implementation in a separate Git branch.

## Learning Outcomes

Through this assignment, I learned that writing a detailed prompt with clear requirements, verification steps, and project constraints produces significantly better AI-generated code than relying on vague instructions. The exercise also reinforced the importance of reviewing AI-generated output instead of assuming it is always correct.
